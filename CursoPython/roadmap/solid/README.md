Resumen de los principios SOLID:
SRP (Single Responsibility Principle): Una clase debe tener una única responsabilidad.
OCP (Open/Closed Principle): El software debe ser extensible sin modificar el código existente.
LSP (Liskov Substitution Principle): Las clases derivadas deben ser intercambiables por sus clases base.
ISP (Interface Segregation Principle): Los clientes no deben depender de interfaces que no usan.
DIP (Dependency Inversion Principle): Las clases de alto nivel no deben depender de clases de bajo nivel, ambas deben depender de abstracciones.
